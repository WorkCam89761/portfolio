Homelabbing

Truenas Scale

Truenas scale is an OS I use to run all the services/containers I run. In order to have them work, I set up a static IP for the host machine (192.168.1.69) so that it doesn't change later on from DHCP causing issues. I also statically put multiple DNS ips (8.8.8.8, 9.9.9.9, 1.1.1.1) so it can resolve domain names, and set the gateway ip to my router (192.168.1.1). I did all of this in the shell or terminal with a script to make sure it stays the same just in case a VM tries to change something that it should not like the ip gateway.

Programs/Scripts

GatewayIP Script

An issue that occured while I was using truenas was that the gateway ip address kept getting changed. As of now I couldn't find the root cause of this, but I already made a fix by creating a .sh file that statically chooses the gateway ip address, its ip address, etc using bash.

For some reason the networking UI that truenas offers did not work which is why I had to resort to using the shell. I also made a crontab which automates when scripts are run to open it everytime it boots and whenever it detects a change in any of the networking settings. Ensuring that the setting stays as it should.

DDNS Script

This script helps automate my DNS provider whenever my public ip address changes using bash. Whenever it changes, the script takes the new public ip and uses Cloudflare's API to update the ip address in order to have all of my services I exposed to the internet continue to work. If I want to add more services in the future, all I have to do is add the service to the subdomain variable and wait a few minutes for everything to update.
